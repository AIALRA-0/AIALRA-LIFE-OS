# Industry Resource Access Strategy

## Principle

Do not wait for commercial tool access to start. The core route must be executable with public resources: Verilator, cocotb, UVM standard, Verification Academy, Yosys, OpenROAD, Tiny Tapeout, RISC-V, Chipyard.

## OPT Status

Likely accessible:
- public/open-source docs and courses
- paid training purchased personally: Doulos, VSD, STEEZY, SoundGym, Berklee, Wharton online, etc.
- Siemens registration/badges if open to individuals
- trial/quote requests for selected commercial tools, subject to vendor terms

Likely difficult without company/university account:
- full Synopsys/Cadence/Siemens/Keysight commercial EDA licenses
- university program tool access if not currently enrolled or if school account expires
- private course labs tied to institutional servers

## Student Status at USC/UCSB/etc.

Potentially improves access to:
- Cadence Academic Network resources through department/lab
- Synopsys University Program through course/lab/license server
- Siemens academic/training programs depending on department agreements
- Keysight ADS through RF/microwave labs if course/lab has license
- professor/lab-specific servers and PDKs

Action:
- Email USC/UCSB ECE graduate advisors and relevant professors with exact access checklist:
  1. Do MS ECE students get Cadence Virtuoso/Innovus/Xcelium access?
  2. Do MS ECE students get Synopsys VCS/Design Compiler/PrimeTime access?
  3. Is access tied to coursework only or general research?
  4. Are international MS students allowed into relevant labs/projects?
  5. Is there a VLSI/ASIC/EDA project course with tool access?

## Company/Internship Status

Strongest access path:
- Join a semiconductor/EDA/AI chip company or lab.
- Use official training from employer portal.
- Get real toolchain exposure, not just course demos.

## Purchase Decision Rule

A paid resource can be purchased only when all conditions hold:

1. It maps to an active skill node.
2. It has a 14-day deliverable.
3. It fills a specific blocker not solvable by public resources.
4. It does not replace building a portfolio artifact.
5. It is logged in the resource library with price, access terms, refund policy and expected output.
